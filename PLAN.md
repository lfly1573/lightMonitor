# 项目开发计划

## 功能模块

### 系统参数配置

    设置全局参数，如：数据保留天数，上报接口密钥。
    设置系统管理用户，可添加多个用户，权限简单划分为管理员和仅可查看数据两类。
    监控数据分组配置，每个分组核心包含分组标记，分组名称，数据获取间隔秒数，被动接收监控字段定义（具体定义规则见监控设置规则），连续几次未能按时获得数据报警设置以及报警方式（可多选）。

### 被动数据接收模块

    用于接收外部数据上报，接受格式为 POST 方式的 json 数据。
    全局使用唯一上报接口，全局定义格式如下：
    ```json
    {
        "group": "分组标记",
        "name": "当前上报数据名称",
        "token": "上报数据要求的密钥",
        "timestamp": "上报时间戳",
        "interval": "当前数据上报间隔秒数",
        "data": {
            "字段1": "值1",
            "字段2": "值2",
            ...
        }
    }
    ```
    返回值：
    ```json
    {
        "code": 0,
        "msg": "success",
        "data": {
            "interval": "要求的上报间隔秒数"
        }
    }
    ```
    特别注意的是主动上报数据的间隔秒数不一定能够和返回值要求的间隔秒数一致，系统判断连续次数时要以上报的间隔秒数为准。
    没有按时上报数据需要报警。


### 主动数据请求模块

    在每个分组下面，可以添加主动查询的url，系统会定时向该url发送请求，获取数据。
    添加内容主要字段为：名称，url，请求方法（GET/POST），请求添加的 header，请求体格式（json/form-data）,返回值字段定义（具体定义规则见监控设置规则），请求间隔秒数。
    主动请求接口如果没有返回数据或者接口请求失败非 200 状态码，也需要报警。

### 报警设置和通知模块

    可以设置多种报警方式，需要默认支持 钉钉机器人 和 telegram 机器人。
    报警设置有全局，分组，以及针对单个分组中的单个名称。

### 数据统计和分析模块

    对每个分组下每个名称可以根据监测的字段进行按时间序的图表展示以及统计和分析，如：平均值，最大值，最小值，中位数等。

## 监控设置规则

    监控设置针对上报的 json 和主动查询返回的 json。均只支持 json 格式。
    支持的字段类型有：字符串，整数，浮点数，布尔值，设置监控字段格式后，如果上报的字段类型与设置的字段类型不一致，系统会自动转换为设置的字段类型。
    通过点号（.）表示字段的层级关系，如："字段1.字段2" 表示字段1下的字段2。
    一个分组下一个名称的监控字段可以定义多个，每个字段可以设置多个不同的报警条件。
    报警条件可以是：大于/小于，等于，不等于，包含，不包含等。
    报警条件还可以是平均值，最大值，最小值，中位数等。
    报警条件可以设置连续多少次才触发报警通知。
    报警可以设置多个方式触发，如：钉钉机器人，telegram 机器人等。
    监控设置可以按组设置，也可以针对单个名称设置。

## 前端要求

- 前端页面基于 vue3 实现并内嵌到程序中，不需要额外部署。
- 前端页面采用蓝白科技风格主题，精简风格。
- 前端页面需要支持多语言，系统内置支持中文和英文，可以方便扩展其他语言，默认为中文。
- 前端采用左右布局，左侧为导航栏，右侧为内容区域。左侧导航栏出了必要的功能模块外，将每个分组作为一个菜单展示，点击分组菜单后，右侧内容区域会展示该分组下的所有名称以及后续设置查看。

## 项目运行目录

- 数据文件放在 `data` 目录下。
- 日志文件放在 `log` 目录下。
- 项目启动时可带参数 -P 指定端口，默认端口8573，启动时检查数据表是否创建，如果没有创建跳转到 /install 页面，设置管理员账号密码并自动创建数据表。
